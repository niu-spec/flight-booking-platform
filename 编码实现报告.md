### 2.2 核心数据模型实现

#### 2.2.1 用户类（User）

对应详细设计报告3.2.1的用户类设计。

**属性实现**：
- `userId` → 继承自AbstractUser的`id`字段
- `phone` → `models.CharField`，手机号字段
- `passwordHash` → 继承自AbstractUser的密码字段

**方法实现**：
- `register()` → 注册账号，设置`is_active`并保存
- `login()` → 返回用户基本信息
- `logout()` → 返回登出标识（实际Token处理在视图层）

**代码位置**：`backend/apps/user/models.py`

#### 2.2.2 航班类（Flight）

对应详细设计报告3.2.2的航班类设计。

**属性实现**：
- `flightNo` → `flight_no`，航班号字段
- `departureTime` → `departure_time`，计划起飞时间
- `basePrice` → `base_price`，基准票价
- `availableSeats` → `available_seats`，剩余座位数

**方法实现**：
- `getAvailableSeats()` → 返回剩余座位数
- `updateStatus(status)` → 更新航班状态（正常/延误/取消）
- `book_seats(count)` → 预订座位
- `release_seats(count)` → 释放座位

**代码位置**：`backend/apps/flight/models.py`

#### 2.2.3 订单类（Order）

对应详细设计报告3.2.3的订单类设计。

**属性实现**：
- `orderId` → `order_id`，订单号
- `totalAmount` → `total_amount`，订单总金额
- `status` → 状态字段（待支付/已支付/已出票/已取消）

**方法实现**：
- `create()` → 创建订单，检查航班可用性并锁定座位
- `cancel()` → 取消订单，释放座位并更新状态
- `confirmPaid()` → 支付成功后更新订单状态
- `issue_ticket()` → 出票操作

**代码位置**：`backend/apps/order/models.py`

#### 2.2.4 支付类（Payment）

对应详细设计报告3.2.4的支付类设计。

**属性实现**：
- `paymentId` → `payment_id`，支付流水号
- `amount` → 支付金额
- `status` → 支付状态（待支付/处理中/成功/失败）

**方法实现**：
- `executePay()` → 发起支付，返回支付网关信息
- `handleCallback(data)` → 处理支付回调，更新订单并出票

**代码位置**：`backend/apps/payment/models.py`

#### 2.2.5 行程类（Itinerary）

对应详细设计报告3.2.5的行程类设计。

**属性实现**：
- `itineraryId` → `itinerary_id`，行程记录标识
- `flightNo` → `flight_no`，航班号
- `departureTime` → `departure_time`，出发时间

**方法实现**：
- `syncFromOrder()` → 从订单同步行程信息
- `refreshStatus()` → 刷新航班动态信息

**代码位置**：`backend/apps/itinerary/models.py`

### 2.3 类关系实现

根据详细设计报告3.3的类关系设计：

- **用户 — 订单（关联，1:0..*）**：通过`Order.user`外键实现
- **订单 — 航班（关联，*:1）**：通过`Order.flight`外键实现
- **订单 — 支付（关联，1:0..1）**：通过`Payment.order`一对一外键实现
- **行程 → 订单（依赖）**：通过`Itinerary.order`外键实现，行程数据由订单驱动

## 4. 业务流程实现

### 4.1 订票主流程实现

对应活动图设计（详细设计报告第4节）：

1. **登录或注册** → 用户通过`/login`或`/register`页面完成
2. **按条件搜索航班** → `FlightSearch.vue`调用`flightApi.search()`
3. **是否有合适航班** → 前端判断航班列表是否为空
4. **选择舱位与座位** → `FlightSearch.vue`弹窗选择舱位
5. **创建订单** → 调用`orderApi.create()`，后端`Order.create()`执行
6. **发起支付** → `OrderDetail.vue`调用`paymentApi.create()`
7. **支付是否成功** → 支付网关回调，后端`Payment.handleCallback()`处理
8. **出票并通知** → 支付成功后自动出票并创建行程

### 4.2 支付出票流程实现

对应顺序图设计（详细设计报告第5节）：

```
用户 → Web前端 → 预订服务 → 支付网关
  |        |           |           |
  |---提交支付--------->|           |
  |        |---调用订单支付------->|
  |        |           |---请求渠道扣款-->|
  |        |           |<--返回扣款结果--|
  |        |<--返回订单处理结果---|
  |<--返回出票结果------|           |
```

实现：
- 用户点击"立即支付" → 前端调用`paymentApi.create()`
- 后端`Payment.executePay()`发起支付 → 返回支付网关信息
- 支付网关异步回调 → `Payment.handleCallback()`处理结果
- 支付成功 → 调用`Order.confirmPaid()`和`Order.issue_ticket()`
- 自动创建行程 → `Itinerary.create_from_order()`

## 5. 部署说明

### 5.1 后端部署

```bash
# 1. 创建虚拟环境
python -m venv venv
source venv/bin/activate

# 2. 安装依赖
pip install -r requirements.txt

# 3. 数据库迁移
python manage.py makemigrations user flight order payment itinerary
python manage.py migrate

# 4. 创建超级用户
python manage.py createsuperuser

# 5. 运行开发服务器
python manage.py runserver
```

### 5.2 前端部署

```bash
# 1. 安装依赖
npm install

# 2. 运行开发服务器
npm run dev

# 3. 构建生产版本
npm run build
```
